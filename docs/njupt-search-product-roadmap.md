# njupt-search 产品判断与信息源路线图

## 我的判断

**njupt-search 可以做，而且比 njupt-exam 的天花板高很多。**
njupt-exam 是单点刚需：只解决“查考试”。njupt-search 可以变成南邮学生的“校园信息入口”：查考试、选课、竞赛、讲座、转专业、奖助学金、社团活动、就业、研究生通知、图书馆开放、停水停电、学院通知，甚至查你的习题讲解、课程资料、项目文档。

但不要一上来就“全站爬南邮”。正确做法是：

**先做高价值、强学生相关、结构稳定的信息源；再扩展到学院站；最后加入个人内容和项目内容。**

# 三、第三优先级：各学院网站

学院站非常值得做，但不建议第一期全接。它们信息分散、样式不统一、重复多、维护成本高。最佳策略是：

**先接计算机、通信、电光、集成电路、自动化、人工智能、物联网、现代邮政/智慧交通这些信息密集学院，再逐步扩展。**

南邮官网教学机构页列出了主要学院站点，包括通信与信息工程学院、电子与光学工程学院、集成电路学院、计算机学院、自动化学院、人工智能学院、物联网学院、理学院、现代邮政学院/智慧交通学院、管理学院、经济学院、外国语学院、教育科学与技术学院、贝尔英才学院、波特兰学院、体育部等。([njupt.edu.cn][8])

学院站适合抓这些内容：

| 类型   | 例子                   | 学生价值 |
| ---- | -------------------- | ---- |
| 学院通知 | 班级事务、毕业设计、实习、答辩、实验安排 | 高    |
| 竞赛通知 | 院赛、校赛、学科竞赛选拔         | 高    |
| 奖助公示 | 奖学金、评优、推免、党员发展       | 高    |
| 讲座活动 | 学术讲座、企业讲座、升学就业分享     | 中高   |
| 项目信息 | 实验室招新、导师项目、科研助理      | 高    |
| 新闻动态 | 学院宣传稿                | 中低   |

建议第一期不要把学院新闻动态混入主搜索默认结果，而是放到“学院动态”过滤器里。否则用户搜“奖学金”时会被一堆会议新闻污染。

# 六、搜索结果怎么过滤“学生相关”

不要只靠关键词。建议每条内容入库时做一个 `student_score`。

## 高权重关键词

这些出现时基本应认为学生相关：

`考试`、`选课`、`成绩`、`四六级`、`重修`、`补考`、`转专业`、`推免`、`毕业设计`、`实习`、`奖学金`、`助学金`、`困难认定`、`宿舍`、`返校`、`离校`、`医保`、`体检`、`停水`、`停电`、`交通管制`、`讲座`、`竞赛`、`大创`、`挑战杯`、`互联网+`、`社团`、`志愿服务`、`招聘`、`宣讲会`、`实习岗位`。

## 低权重或默认过滤关键词

这些可以入库，但默认不推：

`党委理论学习`、`巡察`、`审计`、`干部任免`、`教职工`、`会议纪要`、`采购`、`招标`、`资产处置`、`党建研究`、`行政统计`、`工会活动`。

# 七、六小时爬取一次是否合适？

**合适，但要分源调度。**

不要所有站点都每 6 小时完整爬一遍。建议：

| 源类型              |      频率 | 策略               |
| ---------------- | ------: | ---------------- |
| 教务、学工、团委、创新创业、就业 |  每 6 小时 | 抓列表前 3 页 + 新链接详情 |
| 学院站              | 每 12 小时 | 抓前 1-2 页         |
| 官网通知、新闻          |  每 6 小时 | 抓列表 + 去重         |
| 图书馆、后勤、保卫        | 每 12 小时 | 假期前后可提高频率        |
| 历史归档             |      每周 | 补漏和重建索引          |
| 你的个人资料/视频        |   发布即更新 | webhook 或手动触发    |

每 6 小时的核心不是“全量重爬”，而是：

1. 抓列表页；
2. 提取链接、标题、日期；
3. 跟已有 URL/hash 比较；
4. 新内容才抓详情页；
5. 附件变动才下载；
6. 内容清洗后入索引。

# 八、产品形态建议

## njupt-search 首页

建议不要做成传统搜索框那么冷冰冰。可以做成：

* 顶部搜索框：`搜考试 / 竞赛 / 奖学金 / 讲座 / 宿舍 / 老师 / 项目`
* 热门入口：

  * 考试安排
  * 竞赛报名
  * 奖助公示
  * 转专业/推免
  * 招聘宣讲
  * 图书馆开放
  * 停水停电
* 今日更新：

  * 今天新增 12 条
  * 教务 3 条
  * 学工 2 条
  * 学院 5 条
* 个性化订阅：

  * 我的学院
  * 我的年级
  * 我的专业
  * 我的关键词

## 搜索结果排序

我建议用这个排序：

**相关度 × 学生相关性 × 新鲜度 × 来源权重**

来源权重大概：

1. 本科生院、研究生院、学工处、研工部：最高
2. 团委、创新创业、就业、学院：高
3. 图书馆、后勤、保卫：中高
4. 官网新闻、宣传类文章：中
5. 个人内容、视频、习题：按搜索意图触发，不要硬插

你的内容不要直接“广告式置顶”，而是做成：

> 相关学习资源
> 《高数期末复习题讲解》
> 《C 语言实验题视频讲解》
> 《数据结构课程项目模板》

这样用户不会反感，反而会觉得是搜索结果的一部分。

# 九、技术架构建议

## 轻量版本

适合你现在快速做：

* 爬虫：Python + `requests` + `BeautifulSoup`
* 定时：GitHub Actions cron，每 6 小时
* 存储：JSON 文件 / SQLite
* 搜索：前端 Fuse.js 或 MiniSearch
* 部署：GitHub Pages / Vercel
* 数据文件：`public/index/documents.json`

优点是简单，和 njupt-exam 技术路线一致。

## 成熟版本

如果数据量上来，建议：

* 爬虫：Python Scrapy / Playwright fallback
* 队列：GitHub Actions 或服务器 cron
* 存储：PostgreSQL / SQLite + object storage
* 搜索：Meilisearch / Typesense
* 向量检索：后续再加，不要第一期就上
* 前端：React + Vite
* 后台：FastAPI
* 订阅：邮件 / RSS / 微信推送 / Telegram bot

第一版我建议不要上大后端。先做静态索引，验证需求。

# 十、最容易踩坑的地方

1. **重复内容很多**
   官网、学院、团委、学工处可能互相转载。必须用标题归一化 + 正文 hash 去重。

2. **日期格式混乱**
   有的是 `2026-05-21`，有的是 `05-21`，有的是页面没有年份。需要从列表页、详情页、URL、上下文推断。

3. **附件很重要**
   很多真正有价值的信息在 Word、Excel、PDF 附件里，比如名单、公示、考试安排。必须支持附件下载和解析。

4. **学院站样式不统一**
   不要为每个站写完全独立爬虫。先写“通用南邮站点适配器”，再对特殊站补 selector。

5. **不要爬需要登录的系统**
   智慧校园、教务系统、研究生管理系统这类登录系统不要碰。公开站点够做一个很强的产品。

6. **公众号内容不要强依赖爬取**
   官网和团委站会链接到微信文章，但微信生态反爬和版权问题多。可以收录公开链接和标题，不建议第一期抓全文。

# 十一、我建议的 MVP

**MVP 名称：njupt-search**

第一版只做四个功能：

1. **全校公告搜索**
   收录教务、学工、研究生院、团委、创新创业、图书馆、后勤、保卫。

2. **分类频道**
   考试 / 竞赛 / 奖助 / 就业 / 讲座 / 生活 / 学院 / 研究生。

3. **每日更新页**
   显示最近 24 小时、7 天新增内容。

4. **学习资源推荐位**
   当用户搜课程、考试、题目、实验、竞赛时，展示你的习题、讲解视频、项目模板。

这版如果做得好，已经比南邮官网自带搜索有用得多。

## 最后给你一个定位

njupt-search 不应该只是“南邮公告搜索”。

更好的定位是：

> **南邮学生的信息入口：搜公告、搜考试、搜竞赛、搜讲座、搜项目、搜资料。**
